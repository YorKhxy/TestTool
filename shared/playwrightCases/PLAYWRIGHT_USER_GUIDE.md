## 1. 功能概述

Playwright 测试模式是一款面向**不会写代码的测试人员**的 Web 自动化测试工具，支持：

- ✅ 导入 `.spec.ts` (Playwright 标准格式) 或 `.md` (Markdown 格式) 用例文件
- ✅ 可视化用例管理，支持拖拽排序和按钮调整执行顺序
- ✅ 单条执行或批量执行用例
- ✅ 实时显示执行日志，包括接口请求、响应和断言结果
- ✅ 每次执行自动生成中文 Markdown 格式日志
- ✅ 导出 Excel 报告，支持选择历史记录

## 2. 页面介绍

Playwright 测试模式包含 4 个主要页面：

| 页面 | 路径 | 功能 |

## 3. 导入用例

### 3.1 支持的文件格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| Playwright Spec | `.spec.ts`, `.spec.js` | Playwright 标准测试文件 |
| Markdown | `.md`, `.markdown` | 格式化测试用例文档 |

### 3.2 导入步骤

1. 点击页面右上角的 **「导入用例」** 按钮

2. 在弹窗中选择文件：
   - **拖拽文件**到虚线框内
   - 或点击 **「点击选择文件」** 从电脑选择

3. 选择文件格式（默认自动识别）

4. 点击 **「导入」** 按钮

5. 导入成功后，页面将显示已加载的用例列表

### 3.3 Markdown 用例格式示例

```markdown
### TC_MERCHANT_001 - 商户审批 - 页面加载验证

**优先级**: P0
**分组**: 商户管理

#### 测试步骤

| 步骤 | 操作类型 | 选择器/值 | 说明 |
|------|----------|-----------|------|
| 1 | navigate | /merchant/approval | 导航到商户审批页面 |
| 2 | waitForSelector | h2:has-text("商户审批") | 等待页面标题出现 |
| 3 | assert | table thead th:has-text("公司名称") | 验证表格表头存在 |
| 4 | assert | span:has-text("共 24 条记录") | 验证总记录数显示 |
```

### 3.4 支持的步骤类型

| 类型 | 说明 | 参数 |
|------|------|------|
| `navigate` | 导航到页面 | `value`: URL 路径 |
| `click` | 点击元素 | `selector`: CSS 选择器 |
| `fill` | 填写输入框 | `selector`: 选择器, `value`: 输入值 |
| `select` | 选择下拉选项 | `selector`: 选择器, `value`: 选项值 |
| `wait` | 等待 | `value`: 毫秒数 |
| `waitForSelector` | 等待元素出现 | `selector`: 选择器 |
| `assert` | 断言验证 | `selector`: 选择器, `options.expected`: 预期值, `options.operator`: 操作符 |
| `screenshot` | 截图 | `selector` (可选): 元素选择器 |
| `evaluate` | 执行 JS | `value`: JS 代码 |
| `extract` | 提取数据 | `selector`: 选择器, `options.extractVar`: 变量名 |

## 4. 管理用例

### 4.1 查看用例详情

- **点击表格任意行**：右侧滑出用例详情面板
- **详情面板内容**：
  - 用例 ID
  - 用例标题
  - 分组
  - 优先级
  - 执行步骤列表（可展开查看每步详情）

### 4.2 调整执行顺序

#### 方式一：按钮调整

1. 在「排序」列找到需要调整的用例
2. 点击 **↑ 上移** 或 **↓ 下移** 按钮
3. 执行顺序将实时更新

```
排序： [↑][1][↓]   ← 点击上移/下移调整位置
       [↑][2][↓]
       [↑][3][↓]
```

#### 方式二：拖拽排序（开发中）

拖拽表格行可自由调整顺序。

### 4.3 选择用例执行

- **单选**：点击行的复选框
- **全选**：点击表头的「全选」按钮
- **批量选择**：配合 Ctrl/Cmd 键多选

### 4.4 删除用例

1. 点击用例行打开详情面板
2. 点击右下角 **「删除」** 按钮
3. 确认删除后，用例将从列表移除

## 5. 执行用例

### 5.1 单条执行

1. 在用例列表中找到目标用例
2. 点击该行最右侧的 **「运行」** 按钮
3. 按钮变为 **「运行中...」** 状态
4. 执行完成后，状态列显示 ✅ 通过 或 ❌ 失败

### 5.2 批量执行

1. 选择多个用例（勾选复选框）
2. 点击页面顶部的 **「批量执行」** 按钮
3. 执行日志窗口自动打开
4. 实时显示每个用例的执行状态

### 5.3 执行日志

执行过程中，页面右下角会显示 **「执行日志」** 面板：

```
┌─────────────────────────────────────────┐
│ 执行日志              [正在执行...]     │
├─────────────────────────────────────────┤
│ [🔄] TC_MERCHANT_001 - 页面加载验证    │
│     4/4 步骤通过 • 2.35s               │
│   ├─ ✅ navigate /merchant/approval     │
│   ├─ ✅ waitForSelector h2              │
│   ├─ ✅ assert 公司名称                │
│   └─ ✅ assert 共 24 条记录            │
├─────────────────────────────────────────┤
│ [🔄] TC_MERCHANT_002 - 按公司名称搜索  │
│     运行中...                           │
└─────────────────────────────────────────┘
```

#### 日志内容说明

每个执行步骤会显示：

| 信息 | 说明 |
|------|------|
| 步骤类型 | navigate / click / fill / assert 等 |
| 描述 | 步骤说明 |
| 状态 | ✅ 通过 / ❌ 失败 / 🔄 运行中 |
| 耗时 | 执行时长（秒） |

**展开步骤详情可以看到**：

- **请求信息**：URL、Method、Headers、Body
- **响应信息**：状态码、响应内容
- **断言结果**：预期值、实际值、操作符、是否通过
- **错误信息**：失败原因

### 5.4 断言操作符

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `==` | 等于 | 预期值与实际值完全相等 |
| `!=` | 不等于 | 预期值与实际值不相等 |
| `contains` | 包含 | 实际值包含预期字符串 |
| `startsWith` | 开头是 | 实际值以预期字符串开头 |
| `endsWith` | 结尾是 | 实际值以预期字符串结尾 |

## 6. 查看报告

### 6.1 访问报告页面

点击顶部导航的 **「报告」** 进入报告列表页。

### 6.2 报告列表

| 列 | 说明 |
|----|------|
| 执行ID | 每次执行的唯一标识 |
| 套件名称 | 测试套件名称 |
| 执行时间 | 开始执行的时间 |
| 耗时 | 总执行时长 |
| 总数 | 用例总数 |
| 通过 | 成功通过的用例数 |
| 失败 | 执行失败的用例数 |
| 状态 | 整体状态（通过/失败） |
| 操作 | 导出、删除按钮 |

### 6.3 刷新报告列表

点击页面右上角的 **「刷新」** 按钮更新列表。

## 7. 导出报告

### 7.1 导出 Excel

1. 在报告列表找到需要导出的记录
2. 点击该行的 **「导出」** 按钮
3. 浏览器自动下载 `.xlsx` 文件

### 7.2 Excel 报告结构

导出的 Excel 包含以下工作表：

| 工作表 | 内容 |
|--------|------|
| **执行概况** | 执行基本信息（ID、时间、耗时、总数等） |
| **用例详情** | 所有用例的执行状态、耗时、错误信息 |
| **步骤_XXX** | 每个用例的详细步骤（每个用例一个工作表） |

### 7.3 报告字段说明

**执行概况**：

| 字段 | 说明 |
|------|------|
| 执行ID | run_时间戳 |
| 套件名称 | 测试套件名称 |
| 开始时间 | 格式：YYYY-MM-DD HH:mm:ss |
| 结束时间 | 格式：YYYY-MM-DD HH:mm:ss |
| 执行时长(秒) | 总耗时 |
| 总用例数 | 用例总数 |
| 通过数 | 成功数 |
| 失败数 | 失败数 |
| 跳过数 | 跳过数 |
| 浏览器 | chromium / firefox / webkit |
| BaseURL | 测试地址 |

**用例详情**：

| 字段 | 说明 |
|------|------|
| 用例ID | 用例唯一标识 |
| 用例标题 | 用例名称 |
| 分组 | 所属分组 |
| 优先级 | P0 / P1 / P2 / P3 |
| 状态 | passed / failed / skipped |
| 开始时间 | 执行开始时间 |
| 结束时间 | 执行结束时间 |
| 耗时(秒) | 单用例耗时 |
| 总步骤数 | 步骤总数 |
| 通过步骤 | 成功步骤数 |
| 失败步骤 | 失败步骤数 |
| 错误信息 | 失败原因（如有） |

**步骤详情**：

| 字段 | 说明 |
|------|------|
| 用例ID | 所属用例ID |
| 用例标题 | 所属用例标题 |
| 步骤序号 | 步骤编号（从1开始） |
| 步骤类型 | navigate / click / fill 等 |
| 步骤描述 | 步骤说明 |
| 状态 | passed / failed |
| 开始时间 | 步骤开始时间 |
| 结束时间 | 步骤结束时间 |
| 耗时(秒) | 步骤耗时 |
| 请求URL | 请求地址 |
| 请求方法 | GET / POST / PUT 等 |
| 请求Headers | 请求头（JSON格式） |
| 请求Body | 请求体内容 |
| 响应状态码 | HTTP 状态码 |
| 响应Body | 响应内容（截取前500字符） |
| 断言预期 | 预期值 |

## 8. 常见问题

### Q1: 导入用例失败怎么办？

**可能原因**：
- 文件格式不正确（仅支持 .spec.ts, .ts, .md, .markdown）
- 文件内容为空
- 文件损坏

**解决方法**：
1. 检查文件扩展名是否正确
2. 确认文件内容非空
3. 尝试重新保存文件后导入

### Q2: 执行用例时页面打不开？

**可能原因**：
- BaseURL 配置错误
- 目标页面路径不存在
- 网络问题

**解决方法**：
1. 进入「设置」页面检查 BaseURL
2. 确认目标页面 URL 正确

### Q3: 断言失败怎么办？

**可能原因**：
- 预期值与实际值不匹配
- 页面加载时间过长，等待超时
- 选择器定位不到元素

**解决方法**：
1. 查看执行日志中的「断言结果」
2. 对比「预期值」和「实际值」
3. 调整用例中的预期值或增加等待时间

### Q4: 如何导出中文不乱码的 Excel？

系统已内置中文编码处理，确保导出时：
- 使用 UTF-8 编码
- Excel 打开后自动识别中文字符

如仍出现乱码，可尝试：
1. 使用 WPS 打开
2. 在 Excel 中通过「数据」→「自文本」导入并选择 UTF-8 编码

### Q5: 执行日志丢失了怎么办？

每次执行后日志自动保存到服务器，可在「报告」页面查看历史记录。
